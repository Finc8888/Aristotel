# 🏛 Беседы с Аристотелем — Руководство по установке

## Структура проекта

```
aristotel/
├── CLAUDE.md              # инструкции для Claude + оглавление свитков
├── system_prompt.md       # системный промпт (роль Аристотеля)
├── MEMORY.md              # хронология бесед по дням
├── PROGRESS.md            # прогресс по концепциям учения
├── memorable_quotes.md    # фразы, которые друг попросил запомнить
├── metaphysics.md         # текст Метафизики в UTF-8 (конвертируется автоматически)
├── today_scroll.md        # свиток на сегодня (создаётся и удаляется автоматически)
├── .session_day           # номер текущего дня (временный, в .gitignore)
├── .gitignore
├── reviews/               # еженедельные обзоры прогресса
│   └── review_YYYY-WNN.md
├── morning.sh             # утренняя подготовка + запуск беседы
├── evening.sh             # завершение беседы + сохранение памяти
├── weekly.sh              # еженедельный анализ прогресса
└── .claude/
    ├── mcp.json           # конфигурация MCP серверов
    └── settings.json      # разрешения для агентов (без запросов одобрения)
```

## 1. Установка Claude Code

```bash
npm install -g @anthropic-ai/claude-code
claude login
```

---

## 2. Конвертация файла Метафизики

Если твой файл закодирован в Windows-1251 (стандарт для русских текстов из интернета),
сконвертируй его один раз перед первым запуском:

```bash
cd ~/aristotel
iconv -f windows-1251 -t utf-8 Aristotel_Metaphisics.txt -o metaphysics.md
```

После этого `metaphysics.md` будет читаться агентами без проблем.
Исходный `.txt` файл можно оставить — он не мешает.

> **Примечание:** `morning.sh` проверяет наличие `metaphysics.md` автоматически
> и конвертирует `.txt` файл сам, если `metaphysics.md` ещё не существует.

## 3. Настройка MCP серверов

MCP даёт агентам прямой доступ к файлам проекта и Git без ручных путей.

### Filesystem-сервер (npm)

```bash
npm install -g @modelcontextprotocol/server-filesystem
```

### Git-сервер (Python / uvx)

Git-сервер — это **Python-пакет**, он не устанавливается через npm.

```bash
# Установи uv (если ещё нет)
curl -LsSf https://astral.sh/uv/install.sh | sh

# Проверь что uvx доступен
uvx --version

# Больше ничего не нужно — uvx скачает mcp-server-git при первом запуске автоматически
```

### Подключение конфига

```bash
mkdir -p ~/aristotel/.claude
cp mcp.json ~/aristotel/.claude/mcp.json
```

Открой файл и замени заглушку на свой реальный путь:
```
/ЗАМЕНИ/НА/ПУТЬ/К/aristotel  →  /Users/твоё_имя/aristotel
```

## 4. Разрешения для агентов (убрать запросы одобрения)

По умолчанию Claude Code запрашивает подтверждение на запись файлов и git-команды.
Чтобы агенты работали автоматически без остановок, положи `settings.json` в папку `.claude/`:

```bash
cp settings.json ~/aristotel/.claude/settings.json
```

Файл разрешает агентам: читать файлы, писать файлы, выполнять git-команды — без запросов.

> **Альтернатива:** флаг `--dangerously-skip-permissions` уже встроен во все скрипты
> как запасной вариант на случай если `settings.json` не сработает.

## 5. Настройка скриптов

```bash
mkdir -p ~/aristotel/scripts
cp morning.sh evening.sh weekly.sh ~/aristotel/scripts/
chmod +x ~/aristotel/scripts/*.sh
```

Открой каждый скрипт и укажи свои пути в секции `НАСТРОЙКИ` в начале файла:

```bash
PROJECT_DIR="$HOME/aristotel"                  # ← путь к репозиторию
METAPHYSICS_FILE="$PROJECT_DIR/metaphysics.md" # ← текст Метафизики в UTF-8
```

Добавь алиасы в `~/.zshrc` или `~/.bashrc`:

```bash
alias aristotel-morning="~/aristotel/scripts/morning.sh"
alias aristotel-evening="~/aristotel/scripts/evening.sh"
alias aristotel-weekly="~/aristotel/scripts/weekly.sh"
```

Перезагрузи конфиг терминала:

```bash
source ~/.zshrc   # или source ~/.bashrc
```

## 6. Ежедневный флоу

### Утром — запустить беседу:
```bash
aristotel-morning
```
Агент: проверит кодировку Метафизики → подтянет GitHub → покажет последнюю память →
подготовит `today_scroll.md` → откроет сессию с Аристотелем.

### В беседе — триггерные команды:

| Команда | Что делает |
|---------|-----------|
| `[СВИТОК]` | Аристотель разбирает today_scroll.md: мысль образно → связь с памятью → пример → вопрос |
| `[ПРОВЕРКА]` | Выбирает концепцию со статусом 👁 из PROGRESS.md и проверяет понимание |
| `[ПОВТОРИ]` | Кратко воспроизводит суть последней беседы из MEMORY.md |
| `[КАРТА]` | Показывает все концепции из PROGRESS.md с их статусами |
| «запомни это» | Аристотель немедленно записывает цитату в `memorable_quotes.md` |

### После заката — завершить беседу:
```bash
# Сначала выйди из Claude Code: /exit или Ctrl+C
aristotel-evening
```
Агент: прочитает номер дня из `.session_day` → проверит, не добавил ли уже Claude запись в MEMORY.md →
обновит MEMORY.md (если нужно) → обновит PROGRESS.md → удалит `today_scroll.md` и `.session_day` →
сделает git commit → запушит на GitHub.

> **Защита от дублей:** если Claude уже записал День N в ходе диалога, evening.sh это обнаружит и не создаст День N+1 поверх. Номер дня всегда берётся из `.session_day`, который morning.sh записывает до начала сессии.

### Раз в неделю — обзор прогресса:
```bash
aristotel-weekly
```
Агент проанализирует 7 дней, составит отчёт и сохранит в `reviews/`.

## 7. Синхронизация с Claude.ai проектом

После каждого `evening.sh` все изменения уже на GitHub.
В Claude.ai проекте нажми «Синхронизировать» — и следующая веб-беседа
начнётся с актуальной памятью.

## 8. Советы по экономии лимитов Pro ($20)

- **Одна беседа = одна сессия.** Не затягивай дольше 15–20 обменов — контекст текущего диалога тоже расходует лимит.
- **Свиток — один раз в начале.** Не вставляй текст повторно в процессе беседы.
- **MEMORY.md чисти раз в месяц.** Удаляй записи старше 20 дней — они уже не нужны для связности.
- **PROGRESS.md не разрастается.** Каждая концепция живёт в одном блоке и меняет статус на месте.
- **Файлы свитков не хранятся в проекте.** `today_scroll.md` существует только во время беседы — это ключевая оптимизация контекста.

## 9. Устранение частых проблем

**Агент спрашивает разрешение на каждое действие:**
Убедись что `settings.json` лежит в `~/aristotel/.claude/settings.json`.

**`today_scroll.md` не создался:**
Проверь что `metaphysics.md` существует и читается: `head -5 ~/aristotel/metaphysics.md`.
Если выводится кракозябра — выполни конвертацию из шага 2.

**В MEMORY.md появился лишний день:**
Такое может случиться если `evening.sh` запустили без предварительного `morning.sh` (нет `.session_day`).
Удали лишнюю запись из MEMORY.md вручную. В следующий раз запускай сессию только через `morning.sh`.

**Ошибка `uvx: command not found`:**
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.zshrc
```

**Ошибка `npm install @modelcontextprotocol/server-git` — Not found:**
Это нормально — такого npm-пакета не существует. Git-сервер работает через `uvx`, не npm.
Следуй инструкции из шага 3.
